一、什么是Promise

从语法说，promise是一个对象，从它可以获取异步操作的结果。
本质上 Promise 是一个函数返回的对象，我们可以在它上面绑定回调函数，这样我们就不需要在一开始把回调函数作为参数传入这个函数了。

promise有三种状态：

  • _待定（pending）_：初始状态，既没有被兑现，也没有被拒绝。
  • _已兑现（fulfilled）_：意味着操作成功完成。
  • _已拒绝（rejected）_：意味着操作失败。
    状态不受外界影响，只有异步操作的结果可以决定当前的状态。
    状态一旦改变，就不会再变，此时就叫resolved（已定型）。

缺点：

  • 创造promise实例后，它会立即执行，无法中途取消。
  • 如果不设置回调函数，Promise内部抛出的错误，不会反应到外部。
  • 处于pending状态时，无法得知目前进展到哪一个阶段（刚刚开始还是即将完成）。

二、如何使用？

1. 创建promise对象

Promise的构造函数接收一个参数：函数，并且这个函数需要传入两个参数：

  • resolve ：异步操作执行成功后的回调函数，把promise的状态从“未完成”变为“成功”
  • reject：异步操作执行失败后的回调函数，把promise的状态从“未完成”变为“失败”

3. Promise.prototype.then()

链式调用
then方法返回的是一个新的promise实例，所以可以在then后面再接一个then。第一个回调函数完成后，返回结果作为参数，传入第二个回调函数。

4. Promise.prototype.catch()

Promise.prototype.catch()方法是.then(null, rejection)或.then(undefined, rejection)的别名，用于指定发生错误时的回调函数。

如果promise错误，会调用catch指定的回调函数；如果then的回调函数中抛出错误，也会被catch捕获。

catch可以捕获它前面所有方法抛出的错误。但catch后面抛出的错误，不会被捕获。

5. Promise.prototype.finally()

不管promise最后的状态，在执行完then或catch指定的回调函数以后，都会执行finally方法指定的回调函数。

6. Promise.all()

Promise.all()方法用于将多个 Promise 实例，包装成一个新的 Promise 实例。

p的状态由p1、p2、p3决定，分成两种情况：

  • 只有p1、p2、p3的状态都变成fulfilled，p的状态才会变成fulfilled，此时p1、p2、p3的返回值组成一个数组，传递给p的回调函数。
  • 只要p1、p2、p3之中有一个被rejected，p的状态就变成rejected，此时第一个被reject的实例的返回值，会传递给p的回调函数。

因为只要有一个报错，Promise.all()就会报错，我们并不能知道后面的请求有没有结束，是成功还是失败。

如果作为参数的 Promise 实例，自己定义了catch方法，那么它一旦被rejected，并不会触发Promise.all()的catch方法，而是触发它自己的catch。

7. Promise.race()

只要p1、p2、p3中有一个实例率先改变状态，p的状态就跟着改变。

8. Promise.allSettled()

Promise.allSettled()方法接受一个数组作为参数，数组的每个成员都是一个 Promise 对象，并返回一个新的 Promise 对象。
只有等到参数数组的所有 Promise 对象都发生状态变更（不管是fulfilled还是rejected），返回的 Promise 对象才会发生状态变更。
Promise.allSettled()的状态只会是fulfilled。

当 Promise.allSettled 完成时，它的 .then() 回调会收到一个 结果数组，数组中的每个元素对应原始 Promise 的最终状态。每个结果对象的结构如下：

状态 对象结构
fulfilled { status: 'fulfilled', value: <结果值> }
rejected { status: 'rejected', reason: <错误原因> }

9. Promise.any()

与promise.race()相似，只要参数实例有一个变成fulfilled状态，包装实例就会变成fulfilled状态；如果所有参数实例都变成rejected状态，包装实例就会变成rejected状态。
所有参数p变为rejected，promise.any()才结束并变为rejected。

10.Promise.resolve()

将现有对象转换为promise对象。

参数分四种情况：

  1. 参数是一个promise实例，直接返回这个实例
  2. 参数是thenable对象，就是有then方法的对象，promise.resolve()把这个对象转为promise对象，并且立即执行它的then方法。
  1. 参数是不具有then方法的对象，或者不是对象。返回新的promise实例，并且立即成为resolved状态，立即执行回调函数
  2. 无参数，直接返回一个resolved状态的promise实例
    立即resolve()的 Promise 对象，是在本轮“事件循环”（event loop）的结束时执行，而不是在下一轮“事件循环”的开始时。

11. Promise.reject()

Promise.reject(reason)方法也会返回一个新的 Promise 实例，该实例的状态为rejected。
